# 计划：输出内容少于一页时自动跳过 less

## 目标
当 SQL 查询输出的行数少于终端高度时，直接输出而不调用 `less -R`。

## 修改文件
- `src/main.rs` — 修改 `print_output` 方法（第981行）

## 实现方案

在 `print_output` 方法中：
1. 获取终端高度（行数）：`term_size::dimensions().map(|(_, h)| h).unwrap_or(24)`
2. 计算输出内容的行数：`output.lines().count()`
3. 比较：如果 `output_lines < term_height`，跳过 less，直接 `print!`
4. 否则正常调用 `less -R`

```rust
fn print_output(&self, output: &str) {
    match &self.pager_command {
        Some(cmd) => {
            let term_height = term_size::dimensions()
                .map(|(_, h)| h)
                .unwrap_or(24);
            let line_count = output.lines().count();

            if line_count >= term_height {
                // 输出超过一页，使用 pager
                use std::io::Write;
                use std::process::{Command, Stdio};
                if let Ok(mut child) = Command::new("sh")
                    .args(["-c", cmd])
                    .stdin(Stdio::piped())
                    .stdout(Stdio::inherit())
                    .spawn()
                {
                    if let Some(ref mut stdin) = child.stdin {
                        let _ = stdin.write_all(output.as_bytes());
                    }
                    let _ = child.wait();
                }
            }
            print!("{}", output);
        }
        None => {
            print!("{}", output);
        }
    }
}
```

## 关键点
- `line_count >= term_height`：等于时也走 less（刚好填满一页也算"多"）
- 保留原有的 `print!("{}", output)` — 这是有意设计，退出 less 后终端历史中仍能看到输出
- 使用已有的 `term_size` 依赖，无需新增依赖

## 验证方式
1. 编译：`cargo build`
2. 运行程序，连接数据库
3. 执行少量数据的 SQL（如 `SELECT 1`），确认不调用 less
4. 执行大量数据的 SQL（如 `SELECT * FROM 大表`），确认调用 less
5. 执行 `pager none` 后再测试，确认无 pager 时行为不变
